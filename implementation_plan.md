# 타이틀 바(Title Bar) 일체감 개선 디자인 방안

현재 런처 내부는 '토스(Toss)' 스타일의 부드럽고 모던한 UI가 적용되어 있으나, 상단 타이틀 바는 Windows OS의 기본 스타일(FormBorderStyle.Sizable)을 따르고 있어 시각적인 단절이 발생하고 있습니다.

이를 내부 UI와 일체감 있게 만들기 위한 3가지 디자인/기술적 접근 방안을 제안합니다.

## Option A: 완전한 커스텀 타이틀 바 (가장 일관된 디자인)
OS 기본 타이틀 바를 완전히 제거하고(FormBorderStyle.None), WinForms 패널과 커스텀 버튼을 이용해 런처 상단을 직접 그리는 방식입니다.

- **디자인 방향**: 
  - 타이틀 바의 배경색을 런처의 메인 배경색(예: 옅은 회색 또는 다크 모드 색상)과 100% 동일하게 일치.
  - 타이틀 바 영역과 컨텐츠 영역의 경계를 허물어 하나의 매끄러운 카드로 보이게 연출.
  - 최소화, 최대화, 닫기 버튼을 모던한 SVG 아이콘이나 부드러운 호버 애니메이션이 적용된 버튼으로 직접 구현.
- **장점**: Windows 10과 Windows 11 모두에서 동일하게 완벽한 '토스 스타일'의 일체감을 보장합니다. 상단 영역에 탭이나 프로필 아이콘 등을 자유롭게 배치할 수 있습니다.
- **단점**: Windows 11의 고유 기능인 '스냅 레이아웃(최대화 버튼에 마우스를 올렸을 때 창 분할 메뉴가 뜨는 기능)'이 기본적으로 작동하지 않으며, 창 가장자리 크기 조절 등을 수동으로 처리해야 합니다.

## Option B: Windows 11 네이티브 타이틀 바 색상 변경 (기본 UX 유지)
OS의 기본 타이틀 바를 유지하되, Windows 11의 DWM(Desktop Window Manager) API를 호출하여 타이틀 바의 색상을 런처 배경색과 강제로 일치시키는 방식입니다.

- **디자인 방향**: 
  - DWMWA_CAPTION_COLOR 속성을 사용하여 타이틀 바 색상을 런처 내부와 완벽하게 동일한 색으로 동기화.
  - 텍스트 색상(DWMWA_TEXT_COLOR)을 다크/라이트 테마에 맞게 조정.
- **장점**: Windows 11의 스냅 레이아웃, 네이티브 그림자, 부드러운 창 크기 조절 및 애니메이션 등 운영체제의 고유 UX를 100% 유지하면서 일체감을 줄 수 있습니다.
- **단점**: Windows 11에서만 완벽하게 작동합니다. Windows 10 사용자의 경우 OS 한계로 인해 색상 변경이 적용되지 않아 기존과 동일한 흰색/테마색 타이틀 바가 표시됩니다.

## Option C: 하이브리드 확장 타이틀 바
기본 창 프레임을 유지한 상태로, 내부 클라이언트 영역을 타이틀 바 위까지 확장(DwmExtendFrameIntoClientArea)하여 타이틀 바 영역을 직접 그리는 방식입니다.

- **디자인 방향**: 
  - 버튼(최소/최대/닫기)은 OS의 것을 그대로 둔 채, 배경만 커스텀 패널로 덮어쓰기.
- **장점**: 스냅 레이아웃이 유지되면서도 타이틀 바 배경을 커스텀할 수 있습니다.
- **단점**: WinForms 환경에서 구현 시 화면 배율(DPI) 문제나 창 크기 조절 시 깜빡임(Flickering) 현상을 잡기가 기술적으로 매우 까다롭습니다.
